《Web应用基础》课程结业报告

1.顶部导航栏

该导航栏是一个相应式的，采用了固定定位，将一个弹性容器置于页面的顶部，容器里面总体分为两个弹性元素，左边的元素是一个symbol引用，symbol引用是一种全新的使用方式，也许这种方式是未来的主流，这种用法是做了一个SVG的集合，支持了多色图标，我还了解到了SVG是矢量缩放图，怎么缩放多不会失真的图。 右边元素是导航栏的主体，采用了媒体查询，当屏幕宽度小于854px时，文字就会隐藏，导航栏的图标就会显示，此时遇到了难题,不知如何实现，点击一次下拉菜单框出现，再点击一次下拉菜单框消失的效果。 后来知晓，通过先实现一个下拉菜单框，此时效果是隐藏着的，配合js中的'toggle()'函数，点击bars图标时，该函数触发，如果该下拉菜单没有'open'类，则为下拉菜单的类添加'open'类，如果有，则移除，一开始下拉菜单的高度为0，定义了过渡动画效果，当下拉菜单被打开时（添加了open类），它的高度变为180px，就会过渡显示出来。


2.顶部导航栏和左下角返回顶部的跳转功能


实现跳转功能用的是'a'超链接标签，为每一个页面的整体div元素的设置一个id值，如果一个a标签需要跳转到该div元素的位置，则将a标签的href属性的值设置为'#+该div的id值',点击a标签，即可跳转到该div所处的位置。


3.首页效果

整体采用了flex布局，将一个div变为弹性容器，里面再放置两个子元素div，为左侧div定义了一个关键帧动画，效果为，从离父元素bottom：100vh且height：0，到离父元素bottom：0且height：100vh，右边的子元素div同理， 也定义了一个关键帧动画，效果为离父元素top：100vh且height：0，到离top：0且height：100vh。


4.网站整体背景色

起初，网站的整体色调是全粉，后来发现有渐变色，就了解到了linear-gradient（）函数，线形渐变（还有一个径向渐变），该函数主要有三个参数，一个是线形渐变的方向，往哪个方向渐变颜色，另外两个参数为颜色，从什么颜色渐变到什么颜色， 所以我就把网站的整体色调变成了从底部向顶部渐变的效果，起始颜色为 #fad0c4，结束颜色为 #ffd1ff。底部会显示较浅的粉色，并逐渐过渡到顶部的淡紫色。


5.走马灯图片展示

将主容器设置为固定宽度和高度的矩形容器，用于包含移动的图片，移动的容器，是一个与主容器相同大小的容器，其中包含了需要轮播的图片， 为移动的容器定义了一个'move'关键帧动画,使容器内的图片在10秒内以线性的方式水平向左移动，形成轮播效果,这时遇到了问题,向左移动着的子容器，会溢出父容器，后来经查证，给父容器添加一个overflow:hiiden,就可以隐藏掉溢出的部分， 还有移动着的图片，并没有达到预期的轮播的效果，最终，经思考得出，需在图片的后面，复制多几张和前面一样的图片，从而达到轮播的预期效果。


6.3D立体相册

先给一个div标签通过'transform-style: preserve-3d;'，将该容器设置为具有3D效果的容器，还为这个容器定义了一个'rota'的动画，使容器可以无限循环地旋转， 3D立体相册主要有两层，里面小层，通过translate，rotate属性，将各个div进行平移、旋转，再调整宽高形成里面一层小的3D立体相册、外面一层大的立体相册， 还为外层大的3D立体相册设置了一个hover效果，当鼠标悬停在3D立体相册时，外层的六个面会向外扩展。


7.发光斜体标签

具体实现:
(1)使用SVG 创建形状，创建一个 SVG 容器，使用'svg' 元素定义宽度和高度，并设置 stroke（描边颜色）、stroke-width（描边宽度）和 fill（填充颜色）属性,使用 'path'标签元素来定义标签的形状，通过'd' 属性设置形状的路径,使用 'line' 元素来创建边框，设置起点和终点坐标，同时设置 stroke 和 stroke-width 属性
(2)使用动画添加发光效果。使用' @keyframes' 定义一个动画，在关键帧中设置形状的描边偏移量（stroke-dashoffset）和描边样式（stroke-width）等属性,将动画应用于形状元素。
(3)使用 CSS 控制文本样式和位置。使用 CSS 设置字体样式、大小、颜色、行高、字间距等属性。使用 'transform' 属性控制文本的位置和旋转。


8.第二页的闪烁字体


使用了'@keyframes'定义了名为'flicker'的动画，控制文字的闪烁效果，使用了':hover'伪类选择器来设置鼠标悬停时的样式,通过'text-shadow'属性为文字添加阴影效果，实现发光效果,还使用了 Google Fonts 提供的字体样式 'Dancing Script' 和 'Warnes'。 总体效果就是鼠标悬停时产生闪烁发光的文字。


9.折叠卡片


折叠卡片是一个设置了'transform-style: preserve-3d;'具有3D效果的div容器，整体分为封面和封底，封面又有正面和反面，反面通过rotate（）翻转到反面， 为卡片设置了一个hover的效果，使用了'transform'属性和3D变换函数'translateZ()'和'rotateY()'，实现了封面在空间中的旋转和平移效果,给父容器设置transition属性实现封面的平滑过渡效果。 使用了'postion'属性给卡片的各个元素进行定位,还使用了'display: flex'和相关属性，实现元素的垂直居中和内容的垂直分布。 总体效果:实现一个卡片翻转效果,当鼠标悬停在卡片上时，会触发样式变化和动画效果，卡片元素以 3D 旋转的方式进行翻转，并显示不同的内容。


10.卡片里的流光按钮


按钮样式为一个圆形按钮，背景由渐变色组成。使用伪元素'::before'创建了一个模糊的背景层，并通过动画让背景渐变移动，形成流光的效果。当鼠标悬停在按钮上时，流光的动画将速度加快。整体效果是一个具有动态流光效果的按钮,当鼠标悬停时会有更快的动画效果和光标样式的变化。


11.猫咪分类图片展示


在一个div容器上设置grid，使用网格布局来定义容器内子元素的布局方式，可以通过'grid-template-columns'和'grid-template-rows'属性指定网格的列和行,使用了线性渐变来创建容器的背景效果,还对图片进行了处理，通过设置图片的填充方式（object-fit）和位置（object-position）来控制图片在容器中的显示效果。


12.滚动视差效果


起初，网站的构想只是平平无奇地将内容堆叠在一起，后来参考了一些国外做得较好的网站，了解到了滚动视差的概念,滚动视差效果是一种常见的视觉效果，它通过在不同的页面元素之间创建视差来产生动态感,原理是利用不同元素以不同的速度滚动或移动，从而在垂直滚动页面时产生层次感或深度感。 一开始是通过滚轮事件监听视口和元素的交叉状态，但经过多次调试，都无法实现视口与元素达到一定交叉态时，元素浮现的效果，于是放弃了这种做法。后来，了解到了Intersection Observer API，这是一种用于异步监听目标元素与视口交叉状态的接口，它提供了一种高效的方式来检测元素何时进入或离开视口，或者与视口发生交叉的变化。 具体的该API使用一个观察者对象来观察一个或多个目标元素，观察者对象会在目标元素与视口交叉状态发生变化时触发回调函数。

使用 Intersection Observer API 的基本流程:

(1)使用'new IntersectionObserver(callback, options)'创建一个'IntersectionObserver'对象,'callback'是一个回调函数，用于处理目标元素与视口交叉状态变化的事件。

(2)观察目标元素。使用'observe()'方法将目标元素添加到观察者对象中，开始监听其与视口的交叉状态变化。

(3)处理交叉状态变化。当目标元素与视口的交叉状态发生变化时，观察者对象会调用回调函数,回调函数可以根据交叉状态信息来执行相应的操作。

(4)停止观察。如果不再需要观察某个目标元素，可以使用'unobserve()'方法停止观察。


项目总结

1.盒子模型

盒子模型是css中一个非常重要的概念，整体思想就是，只要是标签元素，就可以把它看作是一个盒子，包含内容区域、内边距、边框和外边距四个部分。 在设置外边距和内边距的方向时，记住沿顺时针方向，按'上右下左'的顺序进行对边距的设置。


2.响应式布局

了解到了相对单位。现代的布局，更多采用的是相对单位，而不是绝对单位，相对单位有'rem,vw,vh'等，而'vw'和'vh'单位非常适合用于响应式布局，可以根据视口的大小自动调整元素的尺寸。通过使用 vw 和 vh 单位设置元素的宽度和高度，可以使元素在不同设备和屏幕尺寸下保持一致的比例和布局。 当然还有媒体查询技术，它可以帮助开发者根据不同的设备和条件提供不同的样式，实现响应式设计和优化用户体验。本次网站设计中，遗憾的是没有使用grid布局中更为强大的minmax（）函数,'minmax()'函数可以根据视口大小和网格容器的宽度来自动调整网格项目的大小,从而更加灵活地实现响应式布局。


3.flex布局和grid布局

(1)flex布局

flex通常用于1维布局，常用的属性为，'flex-direction'指定 Flex 项目的排列方向,'justify-content'定义 Flex 项目在主轴上的对齐方式,'align-items'定义 Flex 项目在交叉轴上的对齐方式, 'flex-grow'定义 Flex 项目的放大比例,'flex-shrink'定义 Flex 项目的缩小比例,'flex-basis'定义 Flex 项目的初始大小,通过以上各种属性的灵活配合，可以在一维层次得到较好的布局。


(2)grid布局

在二维层次的布局，更多的是用grid布局，可以将容器划分为行和列的网格，grid布局相比于flex布局，它会更加灵活，且更具有精确性，可以精确控制元素所处的位置，具有'minmax()'函数，可以灵活适应各种屏幕的尺寸大小，从而实现响应式布局。


4.滚动视差

这个知识点，可以说是意外收获，从当下比较流行的一些网站设计理念中了解到的。
